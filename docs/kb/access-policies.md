# Access Policies

## Overview

All tool access requests at the company go through the IT request portal. Most SaaS tools require manager approval; the typical SLA is one business day. Self-service is available for a small set of low-risk tools.

## Figma

Figma access requires manager approval. Submit a request through IT (or via this assistant) and your manager will receive a Slack notification. Once approved, IT will provision a Figma seat within one business day.

## GitHub

GitHub access requires manager approval and Security review for any repos containing customer data. Standard repo access is granted within one business day; sensitive repos may take 3-5 business days.

## Slack

Slack workspace access is provisioned automatically on day one as part of onboarding. Private channels require an invite from a current channel member; channel admins can add or remove people without an IT ticket.

## Jira

Jira access requires manager approval. Specify whether you need read-only, contributor, or admin role. Standard provisioning is one business day.

## AWS Console

AWS console access requires both manager approval and Security review. Specify the account name (e.g. dev, staging, prod) and the role (read-only, developer, admin). Sensitive accounts may require additional training before access is granted.

## Notion

Notion is a self-service tool. New employees can sign in with their company Google account at notion.so. Contact IT only if SSO does not recognize your email.

## Self-service tools

The following tools do not require an IT ticket: Notion, Loom, Calendly, Google Workspace, Zoom (basic). Sign in with SSO and you are good to go.

## Unsupported tools

Tools that are not on the company-approved list cannot be provisioned by IT. Submit a software-evaluation request through Procurement; expect a 1-2 week review for new vendor approval.
